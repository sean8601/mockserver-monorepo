EXPERIMENTAL — HTTP/3 support is an MVP (Minimum Viable Product). It is off by default and must be explicitly enabled. See MVP scope and limitations below.

MockServer can optionally accept HTTP/3 requests over QUIC (UDP). HTTP/3 is the third major version of the HTTP protocol, using QUIC as the transport instead of TCP. This is useful when you want to test clients that connect using HTTP/3.

 

Enabling HTTP/3

HTTP/3 is disabled by default (http3Port defaults to 0). Set it to a non-zero UDP port number to start the listener:

MethodExample
Java system property -Dmockserver.http3Port=8443
Environment variable MOCKSERVER_HTTP3_PORT=8443
Configuration API Configuration.configuration().http3Port(8443)

When http3Port is 0 (the default), the HTTP/3 listener is not started and has no impact on the existing TCP/HTTP server.

 

Native platform requirement

HTTP/3 uses the netty-incubator-codec-http3 library, which requires a native BoringSSL library for the QUIC transport. The native library is included automatically for the following platforms:

  • linux-x86_64
  • linux-aarch_64
  • osx-x86_64
  • osx-aarch_64
  • windows-x86_64

If the native library cannot be loaded on the current platform, the HTTP/3 server will fail to start. The existing TCP/HTTP server is not affected.

 

TLS

The HTTP/3 server generates a self-signed EC certificate at startup using the BouncyCastle library (already a MockServer dependency). The certificate uses ALPN h3 to negotiate HTTP/3. This is suitable for testing. Configurable certificates are a planned follow-up.

 

MVP scope and limitations

The current HTTP/3 implementation is an MVP that proves the transport layer works. Its request handler echoes the request method and path in the response body:

MockServer HTTP/3 echo - method: GET, path: /hello

The following are not yet supported in the HTTP/3 listener:

  • Expectation matching — HTTP/3 requests are not yet routed through the expectation engine. Use the existing TCP/HTTP ports for expectation-based mocking.
  • Proxy mode — HTTP/3 proxy forwarding is not implemented.
  • Configurable TLS certificates — the self-signed certificate is generated at startup and cannot be replaced.
  • Dashboard visibility — HTTP/3 connections do not appear in the MockServer dashboard.
  • Integration with MockServer lifecycle — the HTTP/3 server is not yet wired into MockServer's standard startup and shutdown path; it must be started separately when using the programmatic API.
