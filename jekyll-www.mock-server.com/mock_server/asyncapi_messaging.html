EXPERIMENTAL — The mockserver-async module is an MVP library. It is not yet wired into the MockServer Docker image or REST control plane. See MVP scope and follow-ups below.

MockServer's AsyncAPI broker mocking lets you drive Kafka and MQTT brokers with realistic example messages, all derived directly from an AsyncAPI 2.x or 3.x specification. This is useful when you want to test a consumer service in isolation: instead of running the real producer, MockServer reads your AsyncAPI spec, generates example payloads for each channel, and publishes them to the broker on demand.

 

How it works

The module follows a three-step pipeline:

  1. Parse — AsyncApiParser reads the AsyncAPI document (JSON or YAML, auto-detected) and builds an in-memory model of channels and their message schemas.
  2. Generate — MessageExampleGenerator produces a JSON example payload for each channel. It uses explicit examples from the spec when present; otherwise it synthesises a minimal valid example from the JSON Schema.
  3. Publish — AsyncApiMockOrchestrator sends each payload to the broker via a MessagePublisher adapter (KafkaMessagePublisher or MqttMessagePublisher). You can publish once or on a repeating schedule.
 

Supported AsyncAPI versions

VersionChannel structureExample resolution
AsyncAPI 2.x channels.<name>.publish|subscribe.message.payload Inline payload.example or message.examples[].payload
AsyncAPI 3.x channels.<name>.messages.<msgName>.payload examples[].payload; basic $ref to #/components/messages/<name>

Both JSON and YAML spec formats are accepted. Missing or incomplete structures are tolerated gracefully — channels with no message definition appear with empty examples and are skipped during publishing.

 

Java usage

Add the mockserver-async dependency to your project, then wire the three components together:

import org.mockserver.async.AsyncApiMockOrchestrator;
import org.mockserver.async.asyncapi.AsyncApiParser;
import org.mockserver.async.asyncapi.AsyncApiSpec;
import org.mockserver.async.publish.KafkaMessagePublisher;

// 1. Parse your AsyncAPI spec (from a string, file, or resource)
String specYaml = Files.readString(Path.of("asyncapi.yaml"));
AsyncApiSpec spec = new AsyncApiParser().parse(specYaml);

// 2. Create a publisher pointed at your test broker
KafkaMessagePublisher publisher = new KafkaMessagePublisher("localhost:9092");

// 3. Create the orchestrator and publish once
AsyncApiMockOrchestrator orchestrator = new AsyncApiMockOrchestrator(spec, publisher);
orchestrator.publishAll();

// Or publish repeatedly on a schedule (e.g. every 500 ms)
orchestrator.startPublishing(500);
// ... run your consumer tests ...
orchestrator.stop();

// Always close the publisher to release broker connections
publisher.close();

The channel name in the AsyncAPI spec maps directly to the Kafka topic name (or MQTT topic path). No additional mapping configuration is needed.

 

Supported brokers

BrokerPublisher classChannel mapping
Kafka KafkaMessagePublisher Channel name → Kafka topic
MQTT MqttMessagePublisher Channel name → MQTT topic
 

MVP scope and planned follow-ups

The current release is a library-level MVP. The following capabilities are planned but not yet available:

  • REST control-plane endpoint — no PUT /mockserver/asyncapi endpoint exists yet; the module must be used programmatically from Java.
  • Docker image integration — the module is not bundled into the MockServer Docker image.
  • Consumer/subscriber mocking — only the publish side is implemented; asserting messages received by MockServer as a consumer is not yet supported.
  • Live-broker integration tests — the test suite uses mocked producers; no live Kafka or MQTT broker is required to build or test the module.
  • Advanced AsyncAPI features — broker bindings, security schemes, correlation IDs, and channels with multiple messages beyond the first are not yet supported.
  • Dashboard UI — async messaging state is not yet visible in the MockServer dashboard.
