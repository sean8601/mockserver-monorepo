Bruno

MockServer provides an official Bruno collection for its REST control plane. Use it to create expectations, verify requests, inspect recorded traffic, and manage server state — all from the Bruno client. Bruno is open-source and git-native: the collection lives in the repository as plain .bru files, so it versions alongside the code and needs no account to use.

Open the Collection

The collection lives in the MockServer repository at examples/bruno:

  • Clone the repository, then in Bruno click Open Collection and select the examples/bruno folder.
  • Select the Local environment (top-right). Its baseUrl variable defaults to http://localhost:1080 — change it to point at any MockServer instance.

Quick Start

Start MockServer with Docker:

docker run -d --rm -p 1080:1080 mockserver/mockserver

Then run the collection requests top to bottom:

  1. Expectations → Create expectation — registers GET /hello → 200 Hello World.
  2. Expectations → Call the mocked endpoint — confirms the mock responds.
  3. Verify → Verify request received — asserts MockServer saw the request.
  4. Traffic → Retrieve recorded requests — inspects what was received.
  5. Manage → Reset — clears all expectations and recorded traffic.

What the Collection Covers

FolderRequests
ExpectationsCreate expectation, call the mocked endpoint, retrieve active expectations
VerifyVerify a request was received, verify a sequence
TrafficRetrieve recorded requests, retrieve logs
ManageStatus, clear (by matcher), reset (everything)

This is the Bruno counterpart of the Postman collection — the same requests, the same baseUrl variable. The full control-plane API is documented as an OpenAPI spec on SwaggerHub.
